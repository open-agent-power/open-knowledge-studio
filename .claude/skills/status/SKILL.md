# /status — Knowledge Base Overview

## Purpose

Show a comprehensive overview of the knowledge base state.

## Steps

1. **Run status** — Execute `oks status`
2. **Interpret** — Present: wiki count, domains, tier distribution, quality avg, drafts, raw files, profiles
3. **Highlight issues** — Low coverage, many pending drafts, high raw count
4. **Suggest actions** — Many drafts → `/promote`, many raw → `/ingest`, low quality → `/compile`
