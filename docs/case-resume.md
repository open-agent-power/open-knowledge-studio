# 托管你的简历

*让简历不再是一份每次求职临时拼凑的静态文档，而是一份持续积累、随时可召回的能力知识库。*

大多数人的简历是"事后考古"：要投递了才回忆这几年干过什么，好的成果早忘了细节。换个做法——**把简历当成一个 goal 驱动的知识库来托管**，让每一段经历在发生时就沉淀进来。

## 设 goal

先划边界，简历知识库才不会变成流水账：

- **年 goal**：拿下某类岗位（如"资深后端"）。
- **季/月 goal**：补齐目标岗位要求里你还缺的证据（如"拿出一个高并发实战案例"）。

goal 决定了什么值得沉淀——和目标无关的琐事，自然不进来。

## 收集入 raw/

把持续产生的"能力证据"随手汇入 `raw/`：

- 做完一个项目 → 记下背景、你的角色、量化结果。
- 解决一个难题 → 记下问题、方案、影响。
- 拿到反馈/评审/奖项 → 存原文。

这一步不加工，只保真——就是[每日循环](daily-loop.md)里的收集与入料。

## 审查沉淀 wiki/

在 goal 约束下 `/promote`，把零散证据提炼成可复用的 `wiki/` 记忆：

- **concept**：你掌握的技能/领域。
- **strategy**：你验证有效的做法（可作 STAR 案例）。
- **anti-pattern**：踩过的坑（面试里的"教训"素材）。

每条带上来源链路，日后能追溯到最初的项目记录。

## 召回复用

要投递或面试时，让 AI 带着这份知识库工作：

```bash
oks search "高并发 优化 成果"
```

- 针对某岗位 JD，召回最相关的经历，生成定制版简历要点。
- 面试前，召回某项目的完整细节和当时的决策。
- 因为 active goal 会给召回加权，和目标岗位相关的证据会优先浮现。

{: .note }
简历知识库从不删除旧经历——过时的会衰减、淡出日常召回，但换个方向求职时，它们还在 Git 历史里等你。

## 第一步该做什么

1. 设一个求职 goal。
2. 把**最近一个**项目的成果记进 `raw/`。
3. 提炼成一条 strategy 记忆，搜一次试试。
